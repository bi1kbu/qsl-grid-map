| `qsl-management` | `>=2.3.21` | 强依赖 | 地图数据直接读取其公开网格接口 |

## 3. API 路由

本插件第一版只提供一个 API。

| 方法 | 路径 | 认证要求 | 用途 |
| --- | --- | --- | --- |
| `GET` | `/apis/qsl-grid-map.bi1kbu.com/v1alpha1/map/page` | 匿名可访问 | 返回通联网格地图页面 |

不得存在的接口：

1. `/apis/qsl-grid-map.bi1kbu.com/v1alpha1/grids`
2. `/apis/qsl-grid-map.bi1kbu.com/v1alpha1/settings`
3. `/apis/qsl-grid-map.bi1kbu.com/v1alpha1/cache/refresh`
4. 任意自定义 `POST`、`PUT`、`PATCH`、`DELETE` 写入接口

## 4. 数据来源

| 数据 | 来源接口 | 说明 |
| --- | --- | --- |
| 通联网格 | `/apis/api.qsl-management.bi1kbu.com/v1alpha1/qso-public/grids` | 浏览器直接请求，不通过本插件服务端代理 |

当前固定查询参数：

1. `sceneType（场景类型）= QSO`

本插件不再透传 `dateFrom（开始日期）`、`dateTo（结束日期）`、`grid（网格）`、`limit（明细数量上限）`，也不在前台提供对应筛选面板。

## 5. 短码

| 短码 | 渲染方式 | 说明 |
| --- | --- | --- |
| `[qsl-grid-map]` | iframe 嵌入 `/map/page?embed=1` | 页面渲染逻辑只维护一套 |

短码参数：

```text
[qsl-grid-map]
```

短码不再解析筛选属性，iframe 仅拼接 `embed（嵌入模式）` 与 `eid（嵌入实例标识）`。

## 6. 菜单与权限

第一版不创建后台菜单，不创建角色模板，不创建自定义权限节点。

原因：

1. 无自定义后台管理页。
2. 无私有数据查询。
3. 无自定义写入接口。
4. 只展示 `qsl-management` 已公开数据。
5. 天地图配置使用 Halo 插件设置能力，后台保存复用 Halo Console 既有鉴权。

## 7. 持久化模型

第一版不创建本插件自有业务 Extension 数据模型。

配置持久化：

| 配置项 | 存储机制 | 必填 | 说明 |
| --- | --- | --- | --- |
| `appKey（应用 key）` | Halo `Setting + ConfigMap` | 是 | 用作天地图瓦片 `tk` 参数 |
| `secretKey（密钥）` | Halo `Setting + ConfigMap` | 否 | 当前不输出到前台页面 |
| `defaultCenterGrid（默认中心网格）` | Halo `Setting + ConfigMap` | 是 | 页面初始加载和无数据时使用，默认 `OM89（中国北京）` |
| `defaultZoom（默认缩放级别）` | Halo `Setting + ConfigMap` | 是 | 页面初始加载和无数据时使用，默认 `3` |
| `minZoom（最小缩放级别）` | Halo `Setting + ConfigMap` | 是 | 页面允许用户缩小到的最小缩放级别，默认 `3` |
| `maxZoom（最大缩放级别）` | Halo `Setting + ConfigMap` | 是 | 页面允许用户放大到的最大缩放级别，默认 `8` |
